# mobile_money 가계부 앱 설계

날짜: 2026-07-08
상태: 확정 (사용자 승인 완료)

## 개요

엑셀(구글 시트) 가계부를 그대로 옮긴 듯한 표 형식의 안드로이드 지출 가계부 앱.
Flutter(Dart)로 개발하며, 데이터는 폰에 저장하고 구글 로그인 후 본인 구글
드라이브에 백업/복원할 수 있다.

사용자가 실제로 쓰던 구글 시트 가계부(월별 시트 탭, 노란 헤더, 격자선,
같은 날짜는 첫 행에만 표시)가 시각적 기준이다.

## 범위

- 지출만 기록한다. 수입, 잔액, 예산 기능은 없다.
- 안드로이드 전용. iOS는 범위 밖.
- 사용자는 1명(본인). 공유·멀티유저 기능 없음.

## 화면 구성

### 메인 화면 (장부 표)

- 화면 전체가 스프레드시트 모양의 표.
  - 행 번호 컬럼 + 노란 헤더 행: **날짜 | 카드 | 사용용도 | 상세 | 금액 | 비고**
  - 격자선 표시, 헤더 행은 스크롤해도 상단 고정.
  - 금액은 `₩9,400` 형식, 오른쪽 정렬.
  - 같은 날짜의 거래가 연속되면 날짜는 그 날의 첫 행에만 표시하고
    이후 행은 빈칸으로 둔다.
  - 표 맨 아래(마지막 거래 행 다음)에 해당 월 지출 합계 행을 표시한다.
- 하단 바: 구글 시트처럼 월 탭 (`2025.7 | 2025.8 | 2025.9 …`).
  - 탭을 누르면 해당 월의 거래만 표에 표시.
  - 탭은 거래가 존재하는 달 + 현재 달을 자동 생성한다. 별도 관리 UI 없음.
  - 하단 왼쪽 ☰ 버튼으로 메뉴(분류 관리, 카드 관리, 설정) 진입.
- 하단 오른쪽 `+` 플로팅 버튼: 입력창(바텀시트) 열기.

### 입력 바텀시트

- 항목: 날짜 선택(기본값 오늘), 카드/현금 선택, 사용용도 선택,
  상세내용(자유 입력), 금액(숫자), 비고(자유 입력, 선택사항).
- 저장하면 메인 표에 행이 추가된다 (해당 날짜 위치에 정렬되어 들어감).
- 기존 행을 탭하면 같은 바텀시트가 해당 내용으로 열려 수정할 수 있고,
  삭제 버튼도 여기에 있다. 삭제는 확인창을 거친다.

### 분류(사용용도) 관리 화면

- 기본 분류: 식사, 간식, 선물, 정기결제, 카페, 편의점, 식재료, 기타.
- 추가 / 이름 수정 / 삭제 가능.
- 거래에서 사용 중인 분류를 삭제하려 하면 경고하고, 삭제 시 해당 거래의
  분류는 "기타"로 바뀐다.

### 카드 관리 화면

- 결제수단 목록: "현금"은 기본 제공(삭제 불가), 카드는 자유롭게 등록
  (예: "KB 다담카드").
- 추가 / 이름 수정 / 삭제 가능. 사용 중인 카드 삭제 시 경고 후
  해당 거래의 결제수단은 "현금"으로 바뀐다.

### 설정 화면

- 구글 로그인 / 로그아웃.
- "드라이브 자동 백업" 켜기/끄기 (로그인 필요).
- "지금 백업" / "드라이브에서 복원" 버튼.
- 마지막 백업 시각, 백업 실패 상태 표시.

## 데이터 모델 (SQLite, sqflite)

- `transactions`: id, 날짜(date), 결제수단 id, 분류 id, 상세(text),
  금액(integer, 원 단위), 비고(text, nullable)
- `payment_methods`: id, 이름, 기본 제공 여부(현금)
- `categories`: id, 이름, 기본 제공 여부
- 월 탭 목록은 `transactions`의 날짜에서 파생한다. 별도 테이블 없음.

## 드라이브 백업/복원

- 로그인 없이 앱의 모든 기능(백업 제외)이 동작한다. 오프라인 우선.
- 인증: `google_sign_in`으로 구글 계정 로그인.
  드라이브 권한은 앱 전용 폴더(`drive.appdata`)만 요청 —
  사용자 드라이브의 다른 파일에는 접근 불가.
- 백업 파일: DB 내용 전체를 JSON 하나로 직렬화해 appDataFolder에 업로드.
  파일명 고정(`mobile_money_backup.json`), 항상 덮어쓰기.
- 자동 백업이 켜져 있으면 거래/분류/카드 변경 시마다 업로드를 시도한다.
  실패(오프라인 등)해도 앱 사용을 막지 않으며, 다음 변경 시 또는
  "지금 백업" 시 재시도한다. 설정 화면에 실패 상태를 표시한다.
- 복원: "현재 폰의 데이터를 모두 덮어씁니다" 확인창 후, 드라이브의
  백업 파일을 내려받아 로컬 DB를 통째로 교체한다.
- 충돌 정책: 마지막 백업이 이긴다(last-write-wins). 실시간 다기기
  동기화는 지원하지 않는다.

## 기술 스택

- Flutter 최신 안정 버전, 안드로이드 전용 (minSdk 23 / Android 6.0+)
- 로컬 DB: `sqflite`
- 상태 관리: `Provider` (ChangeNotifier)
- 구글: `google_sign_in`, `googleapis` (Drive v3, appDataFolder)
- 표: ListView 기반 커스텀 렌더링 (고정 폭 컬럼, 직접 그린 격자선,
  고정 헤더). 서드파티 테이블 패키지에 의존하지 않는다.

## 코드 구조

```
lib/
  data/      # DB 열기/마이그레이션, 거래·분류·카드 저장소(repository)
  sync/      # 구글 로그인, 드라이브 업로드/다운로드, 백업 직렬화
  screens/   # 장부(메인), 입력 바텀시트, 분류 관리, 카드 관리, 설정
  widgets/   # 표 행/셀/헤더, 월 탭 바 등 공용 위젯
  models/    # Transaction, Category, PaymentMethod
```

각 계층은 아래 계층만 의존한다: screens → data/sync → models.
화면 코드는 SQL이나 드라이브 API를 직접 다루지 않는다.

## 오류 처리

- 백업 실패: 조용히 기록하고 설정 화면에 상태 표시. 앱 동작에 영향 없음.
- 복원 실패(파일 없음, 네트워크 오류): 로컬 데이터를 건드리지 않고
  오류 메시지만 표시.
- 금액 입력은 숫자만 허용, 0원 이하 저장 불가.

## 테스트

- 단위 테스트: 월별 필터링, 합계 계산, 같은 날짜 첫 행만 표시하는 규칙,
  백업 JSON 직렬화/역직렬화 왕복.
- 위젯 테스트: 표 렌더링(헤더 고정, 날짜 생략 규칙), 입력 바텀시트의
  저장/수정/삭제 흐름.
- 수동 확인: 실제 구글 계정으로 백업/복원, 실기기 설치.
